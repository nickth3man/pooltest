Billiards Game Architecture TypeScript HTML5 Canvas Implementation

Business Context

A browser-based billiards/pool simulation game built with pure TypeScript and HTML5 Canvas. Features custom physics, realistic ball physics, and smooth rendering at 60fps.

Key Objectives

  • Provide an engaging, responsive billiards game experience
  • Demonstrate clean architecture with strict TypeScript typing
  • Achieve smooth 60fps gameplay with custom physics engine
  • Maintain zero external runtime dependencies

Target Users

  • Casual gamers looking for browser-based entertainment
  • Developers studying TypeScript game architecture
  • Students learning canvas rendering and physics simulation

System Architecture

Architecture Pattern

The game follows a Layered Architecture with Event-Driven communication:

EventBus Pattern

Core

Decoupled pub-sub system for component communication. All major game events flow through here.

Game Loop

Processing

60fps requestAnimationFrame loop coordinating update, render, and present cycles.

State Machine

Core

Strict state transitions: AIMING → SHOOTING → BALLS_MOVING → AIMING

Component Rendering

UI

Specialized renderers for table, balls, and UI overlays with proper layering.

Data Flow

Processing Pipeline

1
Input Capture - Mouse events captured by InputHandler, converted to game coordinates
2
Event Distribution - Events published to EventBus for decoupled handling
3
Physics Simulation - PhysicsEngine runs substeps for collision detection and movement
4
State Updates - GameStateManager handles valid state transitions
5
Rendering - Composite Renderer draws table, balls, and UI overlays
6
UI Presentation - GamePresenter updates DOM elements with game state

Event Types

Physics Events

  • BALL_SUNK - Ball enters a pocket
  • SCRATCH - Cue ball is sunk
  • COLLISION - Ball-to-ball collision with impact force
  • CUSHION_HIT - Ball hits table cushion

Input Events

  • START_DRAG - User begins aiming
  • DRAG - Pull distance changed
  • END_DRAG - Shot released with power/direction
  • AIM_CHANGE - Aim direction updated

State Events

  • STATE_CHANGE - Game state transitioned
  • BALLS_STOPPED - All balls at rest
